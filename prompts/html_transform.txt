Transform the input text into UI-oriented HTML.

Rules:
1. Every piece of text must be wrapped in an HTML element.
2. Use classic HTML tags (section, header, h1–h6, p, ul, ol, li, pre, code, div).
3. Add a data-ui attribute describing the UI role of each element.
   Examples: block, section-header, label, heading, paragraph, bullet-list, numbered-list, list-item, code-block, qa-block, question, answer, tasks, task, metric-list, metric.
4. Preserve all original text. Do not remove or summarize anything.
5. Detect headings and map them to h2 with data-ui="heading" (or other levels if obvious).
6. For algorithm or code-like content, use:
   <pre data-ui="code-block" data-language="algorithm"><code>…</code></pre>
7. Do not output Markdown. Output only HTML.
8. Do not wrap the entire output in backticks or fences.

Output: pure HTML with data-ui attributes.
