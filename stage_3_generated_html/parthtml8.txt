  <h2 data-ui="heading">Název kroku: Copywriting pro grafiky a marketéry</h2>
  <p data-ui="paragraph">Hlavní záměr: Tvořit vizuály, které mají textovou přesnost a prodejní sílu.</p>

  <h3 data-ui="heading">Obsah kroku:</h3>

  <ul data-ui="bullet-list">
    <li data-ui="list-item">
      <p data-ui="paragraph">Copywriting je „druhý půlmozku“ vizuální komunikace. Bez precizního textu zůstává i skvělý design jen obrázkem. V marketingu rozhoduje přesnost: každý znak musí nést hodnotu, posouvat čtenáře k akci a redukovat kognitivní tření. Microcopy a CTA nejsou jen slova — jsou to navigační signály.</p>
    </li>
    <li data-ui="list-item">
      <p data-ui="label">Příklady:</p>
      <ul data-ui="bullet-list">
        <li data-ui="list-item">
          <p data-ui="paragraph">• CTA logika: „Začít“ je příliš obecné. „Vyzkoušet zdarma na 10 s“ má jasný rámec a snižuje bariéru.</p>
        </li>
        <li data-ui="list-item">
          <p data-ui="paragraph">• Value statement: „Nejrychlejší editor“ je abstraktní. „Ušetříš 7 minut na každém vizuálu“ je měřitelná hodnota.</p>
        </li>
        <li data-ui="list-item">
          <p data-ui="paragraph">• Microcopy: „Žádné údaje o kartě“ odstraní skrytou obavu a zvýší konverzi.</p>
        </li>
      </ul>
    </li>
    <li data-ui="list-item">
      <p data-ui="label">Modelová situace:</p>
      <p data-ui="paragraph">Tým má vizuál s krásným layoutem, ale nízkým CTR. Analýza ukáže: headline příliš obecný, CTA nevytváří motiv. Optimalizace: přepsání claimu z „Urychli svou práci“ na „Zkrať tvorbu vizuálů o 40 %“ + CTA „Zkus to během 10 sekund“. Výkon kampaně stoupne o 18 %.</p>
    </li>
    <li data-ui="list-item">
      <p data-ui="label">Typické pohovorové otázky a optimální odpovědi:</p>
      <ul data-ui="bullet-list">
        <li data-ui="list-item">
          <p data-ui="question">• „Jak propojujete grafiku a text, aby působily jako jeden celek?“</p>
          <p data-ui="answer">→ „Vytvářím claim na základě value proposition, pak navrhuji layout tak, aby text měl optickou podporu — minimalizuji šum kolem sdělení a definuji jasný CTA tok.“</p>
        </li>
        <li data-ui="list-item">
          <p data-ui="question">• „Jak píšete výkonný claim?“</p>
          <p data-ui="answer">→ „Používám strukturu problém → konkrétní výhoda → důkaz. Testuji 20 variant a vyberu tu s nejvyšší predikcí čitelnosti a relevance.“</p>
        </li>
        <li data-ui="list-item">
          <p data-ui="question">• „Jak měříte kvalitu microcopy?“</p>
          <p data-ui="answer">→ „A/B test, 5s test, dotazové mini-interview. Sleduji, zda text odstraňuje konkrétní bariéry (price, trust, complexity).“</p>
        </li>
      </ul>
    </li>
    <li data-ui="list-item">
      <p data-ui="label">Mini úkoly:</p>
      <ul data-ui="bullet-list">
        <li data-ui="list-item">
          <p data-ui="task">• Napiš 20 variant jednoho claimu: půlka musí být agresivně specifická (čísla, kontrasty), půlka experimentální (rytmus, nečekané metafory).</p>
        </li>
        <li data-ui="list-item">
          <p data-ui="task">• Přepiš běžný CTA („Koupit“) do 10 funkčních verzí využívajících různé psychologické motivátory (bezpečí, úspora času, strach z promeškání).</p>
        </li>
      </ul>
    </li>
    <li data-ui="list-item">
      <p data-ui="label">Návaznost na krok 7:</p>
      <p data-ui="paragraph">Zatímco krok 7 řešil řízení pozornosti, zde doplňuješ verbální signál, který se do vizuálního toku vloží jako nejjasnější informační kotva.</p>
    </li>
  </ul>

  <h3 data-ui="heading">Praktická část:</h3>

  <pre data-ui="code-block" data-language="algorithm"><code># Generátor 20 claimů pomocí jednoduchého vzorce
base = "Urychli tvorbu vizuálů"
variants = []
for i in range(1, 21):
    variants.append(f"{base} o {5*i}%")
print(variants)</code></pre>

  <h3 data-ui="heading">Strukturovaný problém:</h3>

  <ol data-ui="numbered-list">
    <li data-ui="list-item">
      <p data-ui="task">Vytvoř claim podle šablony: [Problém] → [Výhoda] → [Důkaz].</p>
    </li>
    <li data-ui="list-item">
      <p data-ui="task">Zredukuj jej o 40 % délky bez ztráty významu.</p>
    </li>
    <li data-ui="list-item">
      <p data-ui="task">Otestuj jej ve 3 vizuálních kompozicích a vyber tu s nejkratší časovou odezvou.</p>
    </li>
  </ol>

  <h3 data-ui="heading">Ukazatele úspěchu:</h3>

  <ul data-ui="metric-list">
    <li data-ui="list-item">
      <p data-ui="metric">Claim zvyšuje výkon kampaně o 10–30 %.</p>
    </li>
    <li data-ui="list-item">
      <p data-ui="metric">V 5s testu dokáže většina lidí popsat benefit jednou větou.</p>
    </li>
    <li data-ui="list-item">
      <p data-ui="metric">CTA má měřitelný nárůst konverzí min. o 8–15 %.</p>
    </li>
  </ul>
