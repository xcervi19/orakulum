  <h2 data-ui="heading">Název kroku: Analytická přesnost</h2>
  <p data-ui="paragraph">Hlavní záměr: Rozumět tomu, co funguje a proč.</p>

  <h3 data-ui="heading">Obsah kroku:</h3>

  <ul data-ui="bullet-list">
    <li data-ui="list-item">
      <p data-ui="paragraph">Analytická přesnost je schopnost číst výkon kreativ stejně plynule, jako grafik čte kompozici. Výsledkem je, že dokážeš předvídat, které prvky povedou k vyššímu CTR, nižšímu CPC, delšímu watch-time nebo silnější retenci. Kreativy nejsou náhodné – řídí se zákonitostmi lidské pozornosti a chování.</p>
    </li>
    <li data-ui="list-item">
      <p data-ui="label">Příklady:</p>
      <ul data-ui="bullet-list">
        <li data-ui="list-item">
          <p data-ui="paragraph">• CTR vs. CPC: vysoké CTR, ale vysoké CPC může znamenat špatný targeting nebo nekonzistenci mezi vizuálem a landing page.</p>
        </li>
        <li data-ui="list-item">
          <p data-ui="paragraph">• Scroll-stop rate: u Meta Ads je zásadní. První 0,2–0,5 s rozhodnou, zda uživatel zastaví feed.</p>
        </li>
        <li data-ui="list-item">
          <p data-ui="paragraph">• Watch-time a retention grafy: na YouTube/TikToku se sledují mikro-propady. Pokud drop nastane přesně v momentě změny scény, znamená to slabý hook nebo šum.</p>
        </li>
        <li data-ui="list-item">
          <p data-ui="paragraph">• Konverze: dobrý vizuál nemusí znamenat dobrý výkon, pokud claim neodpovídá realitě nebo LP nedrží kvalitu.</p>
        </li>
      </ul>
    </li>
    <li data-ui="list-item">
      <p data-ui="label">Modelová situace:</p>
      <p data-ui="paragraph">Kampaň má CTR 1,8 %, ale konverze je nízká. Analýza ukáže, že vizuál slibuje „Ušetři 40 % času“, ale LP ukazuje náročný onboarding. Výsledek: kognitivní nesoulad. Optimalizace: změna CTA z „Začni teď“ na „Vyzkoušej si demo“ a úprava hero sekce LP. Konverze +27 %.</p>
    </li>
    <li data-ui="list-item">
      <p data-ui="label">Typické pohovorové otázky a optimální odpovědi:</p>
      <ul data-ui="bullet-list">
        <li data-ui="list-item">
          <p data-ui="question">• „Které metriky sledujete jako první a proč?“</p>
          <p data-ui="answer">→ „Scroll-stop rate pro Meta, watch-time a prvních 3 s pro video, CTR pro ověření relevance, konverze pro celkovou kvalitu. Každá metrika odpovídá jiné části funnelu.“</p>
        </li>
        <li data-ui="list-item">
          <p data-ui="question">• „Dokážete určit, proč kreativě klesl výkon?“</p>
          <p data-ui="answer">→ „Ano – používám pattern-matching: změna barvy CTA, slabší kontrast, příliš technický claim, rušivý detail v prvních sekundách.“</p>
        </li>
        <li data-ui="list-item">
          <p data-ui="question">• „Jak byste předpověděl výkon nové kreativy?“</p>
          <p data-ui="answer">→ „Na základě knihovny předchozích kampaní, podobných struktur, vizuální složitosti a kvality claimu. Tvořím scoring model.“</p>
        </li>
      </ul>
    </li>
    <li data-ui="list-item">
      <p data-ui="label">Mini úkoly:</p>
      <ul data-ui="bullet-list">
        <li data-ui="list-item">
          <p data-ui="task">• Vyber 10 reklamních vizuálů a určuj jejich společné vzorce: typ kompozice, síla kontrastu, pozice claimu, CTA logika, barevné páry, množství textu.</p>
        </li>
        <li data-ui="list-item">
          <p data-ui="task">• Vytvoř rating od 1 do 5 pro každý vizuál podle předpokládaného CTR a porovnej s reálnými výsledky (pokud je máš).</p>
        </li>
      </ul>
    </li>
    <li data-ui="list-item">
      <p data-ui="label">Návaznost na krok 9:</p>
      <p data-ui="paragraph">V kroku 9 se řešila adaptace. V kroku 10 se učíš kvantitativně posoudit, co se při adaptaci skutečně osvědčilo a proč.</p>
    </li>
  </ul>

  <h3 data-ui="heading">Praktická část:</h3>

  <pre data-ui="code-block" data-language="algorithm"><code># Jednoduchý scoringový model kreativy
features = {
    "contrast": 0.3,
    "cta_clarity": 0.25,
    "claim_specificity": 0.2,
    "visual_simplicity": 0.15,
    "first_frame_strength": 0.1
}

def predict_score(ad):
    score = 0
    for f, w in features.items():
        score += ad[f] * w
    return score

sample_ad = {
    "contrast": 0.9,
    "cta_clarity": 0.8,
    "claim_specificity": 0.7,
    "visual_simplicity": 0.6,
    "first_frame_strength": 0.8
}

print(predict_score(sample_ad))</code></pre>
